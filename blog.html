Discuss the scope of var, let, and const

var let const
1 Variables declared with var have function scope or global scope, depending on where they are declared. Variables declared with let have block scope. They are only accessible within the block in which they are defined, whether it's a loop, an if statement, or any other block. Variables declared with const are block-scoped, just like variables declared with let.
2 If a variable is declared inside a function, it is only accessible within that function. They are not accessible before they are declared a concept known as the "temporal dead zone." The value of a const variable cannot be re-assigned once it is assigned.
3 var is also subject to hoisting, meaning that the variable can be declared after it is used within the function, and it will still be accessible Redefining a variable with the same name in the same scope will result in an error. For objects and arrays, the properties and elements of the object or array can be modified..

In modern JavaScript development, it's generally recommended to use let and const over var due to their more predictable and less error-prone scoping behavior. Use const when the value of a variable should not be changed, and use let when the value may change.

Tell us the use cases of undefined and null

NULL UNDEFINED
1 Null represents the intentional absence of any object value. It is explicitly assigned to a variable to indicate the absence of a value. In JavaScript, undefined is a variable that has not been assigned a value. It is the default initial value for variables.
2 It is often used as a placeholder when an object doesn't have a value assigned but is expected to eventually hold one. It is also the default return value of functions that do not explicitly return anything.
3 Commonly used to explicitly indicate the absence of a value or to reset a variable. Used when a variable has been declared but not assigned a value.
4 Useful when you want to clear the value of a variable that was previously assigned an object or a value. It can indicate a missing property in an object.

What do you mean by REST API?

A REST (Representational State Transfer) API is a type of web service that allows communication between different systems or applications using a set of predefined resources and HTTP methods (such as GET, POST, PUT, DELETE). It is a widely used architecture pattern for building web services and can be used to expose application functionality to external clients.
